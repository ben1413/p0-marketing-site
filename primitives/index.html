The Primitives

Three primitives.
Everything else is implementation.

Not features. The physics of the system.

THE DOCTRINE
PROMOTE

Atomic commit for decisions. The line between thinking and deciding.

A commitment event with actor, authority, evidence, and timestamp declared at the moment it happens. If it is not promoted, it is allowed to fade.

Every Promote event generates a ledger item, a memory summary index, and an audit trace. Three subsystems connected by one boundary.

Project → Ledger Team → Memory Personal → Working

Promote is not logging. Logging captures what happened. Promote declares what was decided - and who was responsible.

AUTHORITY

Every decision records who was responsible and under what scope.

Delegation is explicit. Scope is declared. Duration is recorded. Project0 does not pretend AI decisions are neutral.

Every authority record carries: actor identity, authority mode, scope boundaries, delegation chain, and timestamp. Nothing is inferred after the fact.

Human-Led Human-in-the-Loop Agent-Autonomous

Human-Led - a human made this call.

Human-in-the-Loop - a human reviewed and committed.

Agent-Autonomous - an agent acted within a declared, scoped delegation.

No mode is assumed. Every mode is declared.

MEMORY

Immutable record of what actually happened.

Ledger - immutable. Append-only. Cannot be edited. Cannot be erased. Brain - mutable. Operational. Supersession is allowed. Silent edits are not.

Promote is the boundary. Core memory is durable and accountable. Working memory is useful and disposable.

EPHEMERAL

Active context window. Exists to reason, not to remember. If not promoted, it fades. Intentional.

BRAIN

Working memory. Mutable via supersession. Where agents and humans collaborate and correct. Not the system of record.

LEDGER

Core memory. Immutable. Append-only. Authority-attributed. Tamper-evident. The institutional spine.

THE CALLS

Five calls.
One governance boundary.

Every operation in Project0 maps to one of these. If you understand the doctrine, the API follows naturally.

run()

Execute an agent with governed context. Every run is persisted. Every trace is attributable.

POST /api/v1/agents/run

promote()

Commit a decision across the governance boundary. The line between thinking and deciding.

POST /api/v1/ledger/promote

delegate()

Grant authority to an agent with explicit scope. Duration recorded. Delegation traceable.

POST /api/v1/authority/delegations

evaluate()

Open a bounded decision window. Close it. Seal it. Hand it to anyone who asks.

POST /api/v1/evaluations

halt()

Stop execution when governance rules trigger. Suspend. Resume. Every trigger recorded.

POST /api/v1/agents/:id/halt

WHY THIS MATTERS

Most systems separate
thinking from doing.
We separate thinking from deciding.

Thinking is cheap. Deciding is consequential. Every agent in Project0 can think - reason, draft, suggest, analyze - without committing anything. The moment it promotes, it has decided. That decision carries a record, an authority declaration, and an audit trail.

That boundary is the system. Everything else is implementation.

Promote is not a feature.
It is a protocol boundary.
